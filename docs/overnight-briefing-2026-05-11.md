## Failures / surprises

### Path-translation bugs (worked around, not source-fixed)

Three of the shell scripts pass POSIX-style paths (e.g. `/c/Users/...`) into Python, which Windows interprets as `\c\Users\...`. The Roboflow SDK then writes data into a `C:\c\Users\...` tree instead of `C:\Users\...`. I moved the files back to canonical locations. **The bugs are still in the scripts.** Affected:

- `scripts/prepare_datasets.sh` → bypassed via `scripts/_bootstrap_build_prepared.py` (existing helper).
- `scripts/download_caries_data.sh` → ran the script anyway, then `Move-Item` from `C:\c\Users\...` to `C:\Users\...`, then patched the auto-generated `data.yaml`'s `path:` field. Files end up in canonical layout.
- After cleanup an empty `C:\c\` directory remains; Remove-Item is sandbox-blocked on `C:\`. Joseph can `rmdir /S C:\c` from cmd.exe to clean up.

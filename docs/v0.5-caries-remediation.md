# v0.5 Caries Pipeline — Remediation TODO

**Status:** Deferred from v0 on 2026-05-11 due to upstream dataset
corruption discovered during hour-0 bootstrap on pickles. The
methodology + code shipped (cleanroom YOLOv8s trainer + adapter +
inference helper + 14 passing tests + orchestrator wiring with
graceful-skip), but the dataset itself is unusable.

## What failed at v0

The pickles-Claude executor at Step 4 of post-bootstrap kickoff
discovered that the only public BW+depth-tier caries dataset under a
permissive license — Renielaz `dental-caries-x-ray` on Roboflow — has a
structurally corrupted class list at the project level (affects every
exported version, including 6190-6199 inclusive).

Roboflow accidentally serialized 6 dataset-description markdown bullets
into the project's class names list:

```
0:  "-"                                                                  (1001 annotations)
1:  "- collaborate with your team on computer vision projects"           (314)
2:  "- collect - organize images"                                        (261)
3:  "Caries - v2 2023-12-22 9-16am"                                      (214)
4:  RA1                                                                  (3)
5:  RA2                                                                  (72)
6:  RA3                                                                  (50)
7:  RB4                                                                  (34)
8:  RC5                                                                  (33)
9:  RC6                                                                  (13)
10: "Roboflow is an end-to-end computer vision platform that helps you" (297)
11: "This dataset was exported via roboflow.com on December 22- ..."    (414)
```

Annotations got drawn against those garbage indices. Result: only 205
of 2978 annotations are real ICCMS-labeled. After our 3-class collapse,
`deep` (RC6) has 13 examples total — too few to train a generalizable
detector regardless of augmentation.

## Pre-flight before any v0.5 caries path

**Run the probe script first** to verify the dataset's actual annotation
distribution, NOT its marketing description:

```bash
python scripts/_probe_roboflow_versions.py        # check available versions
python scripts/_probe_roboflow_version_yaml.py    # check class list integrity
```

(These should be promoted from `_`-prefixed bootstrap-only to permanent
`scripts/diagnostics/` in a follow-up commit.)

The lesson from 2026-05-11: **a dataset's description is not its
ground truth**. Always probe the YOLO/COCO class list AND the
annotation distribution before scoping ML work against a public
dataset.
